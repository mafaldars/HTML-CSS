Nota: As CSS inline é uma técnica que funciona mas não é recomendável porque temos de ir tag a tag para estilizar, demora muito tempo caso o conteúdo do site seja extenso. Imaginemos que contém 100 tags <p> e se tivessemos de estilizar um a um, é incomportável. Manutenção custosa, a técnica não é prática nem eficiente.

Nota: As CSS internal/local são compostas por seletores (body, h1, h2, etc) que contêm declarações (background-color, font-family, etc) e finalizamos (sempre) cada declaração com ; . Desta forma podemos criar os seletores e todos os elementos da página herdaram os valores atribuídos.

Desvantagens: Estilizar elementos ocupa muitas linhas de código, ao termos o estilo no ínicio da página, teriamos de fazer scroll até baixo para encontrar o ínicio do HTML. Se tivessemos 30 páginas no site, teríamos de copiar e colar os estilos em cada página, não é prático.
CSS local é útil quando aplicado a um único documento, apenas.

EX: body { background-color: blanchedalmond; font-family: Arial, Helvetica, sans-serif; font-size: 20px; }

Nota: No mesmo documento podemos usar CSS external, internal e inline, o inline prevalece e sobrepõe-se aos outros, seguido do internal e depois o external, contudo, o external é o mais versátil e mais prático de utilizar.
