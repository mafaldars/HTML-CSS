Imagens dinâmicas

Tente abrir este site em gadgets diferentes ou simplesmente aumente e diminua o tamanho do seu browser.

Atualmente podemos aceder a um site apartir de vários tipos de gadgets: smartphones, tablets, notebooks, computadores desktop, smart TV, etc. Pensando nisto, iremos precisar de adaptar o conteúdo dos sites que iremos criar para as diferentes telas.

As imagens são o tipo de arquivo que mais sofre com os problemas de exibição. Uma TV pode exibir imagens gigantes mas se usarmos a mesma fotografia para ser apresentada num smartphone teremos que redimensionar obrigatoriamente a imagem com as CSS.

Smartphone: Utilizar uma imagem 300x300

Tablet: Utilizar uma imagem de 700x700

TV: Utilizar uma imagem 1000x1000

Nota: o design responsivo é de extrema importância. Por exemplo, se o user entra num site e sai o Google interpreta isso como bounce rate (taxa de rejeição), esta taxa é uma medida para dar valor ou desvalorizar um site.
Auto Bounce rate é quando um user pesquisa por algo, é direcionado para um site mas sai, não permanece, então o Google parará de indicar a outros users o site para aquele artigo procurado.

Nota: Dependendo do browser a indicação na tag <audio> do autoplay deverá reproduzir automaticamente o áudio. Para termos os controles do áudio na tag <audio> devemos colocar controls.

Formatos áudio:

  • MP3 (melhor)
  • WAV (muito pesado, evitar usar)
  • OGG (formato open-source)

Para termos compatibilidade total dos formatos em vários navegadores devemos ter outras opções de reprodução, pois a tag apresentada acima poderá e não é compatível com todos os browsers.

Outra forma de reproduzir áudios

A outra opção de código, dentro da tag <audio> inserimos:
<audio>
<source src="audio.mp3" type="audio/mpeg">
<source src="audio.wav" type="audio/wav">
<source src="audio.ogg" type="audio/ogg">
</audio>

Nota: Lembrar que devemos adicionar os diferentes tipos de áudio na source e no type (WAV ou OGG).
A ordem em que colocamos o código definirá a ordem pela qual o browser irá tentar reproduriz os formatos (de cima para baixo).
Caso o browser não consiga reproduzir mediante as instruções dadas pelo código, infelizmente o browser não conseguirá reproduzir áudios, posto isto, devemos colocar essa indicação (mensagem) na tag <p>(que irá ser exibida para o user) e disponibilizar o arquivo para download com a tag <a href>.

EX: <a href="midia/daynoche.mp3" download="daynoche.mp3" type="application.mp3">
